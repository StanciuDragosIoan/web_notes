JS Introduction

JavaScript is a general purpose programming language. It was initially designed only for frontend interaction but starting with 2009 and the emergence of JS runtimes such as nodeJS it became a server side language too, thus being capable of doing any system/back-end related task as well as many other things outside the browser (e.g. mobile, desktop applications).

the JS console

The console object has a bunch of useful methods for debugging purposes.At the most basic level, we can log variables with it to test their value throughout our program: 

        console.log("a string...");
        console.log([1,2,3]);
        console.log({name: "John Doe", age: 20});

The console.clear(); method clears the output in the console.

We can also use the console.table() method to display data more elegantly: 

      let object = { name: "John", age: 20 };
      let array = [1, 2, 3];
      console.log(object);
      console.log(array);
      console.table(object);
      console.table(array);

Variables in JavaScript

Variables are the building blocks of our programs (together with functions). They are containers for data, and we use them throughout our code. With ES5 and older versions, the only way to declare a variable was with the var keyword. With ES6 and further on, we have 2 new ways to declare a variable: let and const: 

        var name = "Mary";

The var and let keywords are similar, there is a small difference between them (the former is function scope and is hoisted while the latter is block level scope and not hoisted). A disadvantage of the var keyword is that it can overwrite the global scope thus polluting it: 

      //global scope
      var a = 1;

      if (true) {
        //block lvl scope
        var a = 2;//overwrites global scope
      }
      //global scope but...
      console.log(a); //logs 2

On the contrary with letwe do not have this issue: 

      //global scope
      let a = 1;

      if (true) {
        //block lvl scope
        let a = 2;//stays here
      }
      //global scope not polluted...
      console.log(a); //logs 1

let and const are block level scoped so they do not overwrite the global scope (unlike var ).

Concerning const it has a single peculiarity compared to let, namely that we cannot re-assing it. We can reassign var and let but not const. So we can do: 

        var num = 1;
        num = 2;//can do this
        var odd = 3;
        odd = 4;//can do this

However, if we try the below: 

      const secondOdd = 5;
      secondOdd = 6;//cannot do this :/

Data Types in JavaScript

There are 6 data types that are primitive in JS: 

      //undefined 
      console.log(typeof undefined); // 'undefined'
      //boolean
      console.log(typeof true); // 'boolean'
      //number
      console.log(typeof 1234); // 'number'
      //string
      console.log(typeof 'string here'); // 'string'
      //bigint
      console.log(typeof 10n); //'bigint'
      //symbol
      console.log(typeof Symbol()); // 'symbol'

There are also 3 reference type data types 

      //null
      console.log(typeof null); // 'object'
      //Object, Array, Map, Set, etc..
      const obj1 = new Object();
      console.log(typeof obj1); // 'object'
      //function
      let myFunction = ()=> {
        console.log("Yo!");
      }
      console.log(typeof myFunction); // 'function'

In order to check the data type of an object in JavaScript, we use the typeof operator

String methods and concatenation

These are basic ways to work with and manipulate string values: 

      const firstName = "William";
const lastName = "Johnson";
const age = 36;
const str = "Hello there my name is Brad";
const tags = "web design,web development,programming";

let val;

val = firstName + lastName;

// Concatenation
val = firstName + " " + lastName;

// Append
val = "Brad ";
val += "Traversy";

val = "Hello, my name is " + firstName + " and I am " + age;

// Escaping
val = "That's awesome, I can't wait";

// Length
val = firstName.length;

// concat()
val = firstName.concat(" ", lastName);

// Change case
val = firstName.toUpperCase();
val = firstName.toLowerCase();

val = firstName[2];

// indexOf()
val = firstName.indexOf("l");
val = firstName.lastIndexOf("l");

// charAt()
val = firstName.charAt(2);
// Get last char
val = firstName.charAt(firstName.length - 1);

// substring()
val = firstName.substring(0, 4);

// slice()
val = firstName.slice(0, 4);
val = firstName.slice(-3);

// split()
val = str.split(" ");
val = tags.split(",");

// replace()
val = str.replace("Brad", "Jack");

// includes()
val = str.includes("foo");

Template literals (template strings)

Template literals are a nice way to display variables embedded in HTML. They were introduced to make the ES5 'concatenating' way of doing it, much easier: 

      const name = 'John';
const age = 31;
const job = 'Web Developer';
const city = 'Miami';
let html;

// Without template strings (es5)
html = '<ul>' +
       '<li>Name: ' + name + '</li>' +
       '<li>Age: ' + age + '</li>' +
       '<li>Job: ' + job + '</li>' +
       '<li>City: ' + city + '</li>' +
       '</ul>';

function hello(){
  return 'hello';
}

// With template strings (es6)
html = `
  <ul>
    <li>Name: ${name}</li>
    <li>Age: ${age}</li>
    <li>Job: ${job}</li>
    <li>City: ${city}</li>
    <li>${2 + 2}</li>
    <li>${hello()}</li>
    <li>${age > 30 ? 'Over 30' : 'Under 30'}</li>
  </ul>
`;

document.body.innerHTML = html;
