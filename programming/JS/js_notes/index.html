Variables in JavaScript

Variables are the building blocks of our programs (together with functions). They are containers for data, and we use them throughout our code. With ES5 and older versions, the only way to declare a variable was with the var keyword. With ES6 and further on, we have 2 new ways to declare a variable: let and const: 

        var name = "Mary";

The var and let keywords are similar, there is a small difference between them (the former is function scope and is hoisted while the latter is block level scope and not hoisted). A disadvantage of the var keyword is that it can overwrite the global scope thus polluting it: 

      //global scope
      var a = 1;

      if (true) {
        //block lvl scope
        var a = 2;//overwrites global scope
      }
      //global scope but...
      console.log(a); //logs 2

On the contrary with letwe do not have this issue: 

      //global scope
      let a = 1;

      if (true) {
        //block lvl scope
        let a = 2;//stays here
      }
      //global scope not polluted...
      console.log(a); //logs 1

let and const are block level scoped so they do not overwrite the global scope (unlike var ).

Concerning const it has a single peculiarity compared to let, namely that we cannot re-assing it. We can reassign var and let but not const. So we can do: 

        var num = 1;
        num = 2;//can do this
        let odd = 3;
        odd = 4;//can do this

However, if we try the below:

      const secondOdd = 5;
      secondOdd = 6;//cannot do this :/

Switches

Switches allow us to perform a type of if(){}else{} logic in a more elegant way. They are useful when a single value changes multiple times: 

const color = 'yellow';

switch(color){
  case 'red':
    console.log('Color is red');
    break;
  case 'blue':
    console.log('Color is blue');
    break;
  default:
    console.log('Color is not red or blue');
    break;
}

let day;

switch(new Date().getDay()){
  case 0:
    day = 'Sunday';
    break;
  case 1:
    day = 'Monday';
    break;
  case 2:
    day = 'Tuesday';
    break;
  case 3:
    day = 'Wednesday';
    break;
  case 4:
    day = 'Thursday';
    break;
  case 5:
    day = 'Friday';
    break;
  case 6:
    day = 'Saturday';
    break;
}

console.log(`Today is ${day}`);

Function declarations and function expressions

A function declaration is something like:

function greet(firstName = 'John', lastName = 'Doe'){
  return 'Hello ' + firstName + ' ' + lastName;
}

*note that we 'declare' the function with the keyword function.

A function expression on the other hand is something like: 

      const square = function(x = 3){
        return x*x;
      };

We still use the keyword function but we put the function inside a variable called square. This is the expression.

IIFE - Immediately Invoked Function Expression

It is a function that runs as soon as it's declared. It can be used to limit isolate scope: 

      (function(name){
        console.log('Hello '+ name);
      })('IFFEE');

ES6 Promises

Promises handle async operations and 'promise' to do something once that finishes using the <span class="snippet">.then()</span> syntax: 

 
      const posts = [
  {title: 'Post One', body: 'This is post one'},
  {title: 'Post Two', body: 'This is post two'}
];
 

function createPost(post) {
  //create promise which takes resolve and reject as arguments in CB 
  return new Promise(function(resolve, reject){
    setTimeout(function() {
      posts.push(post);

      //error for testing
      const error = false;

      if(error){
        //call reject (promise failed)
        reject('ERROR SOMETHING went wrong');
      } else {
        //call resolve(); 
        resolve();
      }
      
    }, 2000);
  });
  
}


function getPosts() {
  setTimeout(function() {
    let output = '';
    posts.forEach(function(post){
      output += `<li>${post.title}</li>`;
    });
    document.body.innerHTML = output;
  }, 1000);
}

//call getPosts in .then();
createPost({title: 'Post Three', body: 'This is post three'})
 //get posts if promise fulfilled
  .then(getPosts)
  //catch error if promise not fulfilled
  .catch((err)=>{
    console.log(err);
  });

The Fetch API

The Fetch API is a newer standard for handling asynchronous code. It returns a promise. 


      //work with text with Fetch

      function getText() {
        fetch("test.txt")
        //call .then since fetch returns a promise
        .then(function(res){
            //returns another promise
            return res.text();
        })
        //call .then() again from the res.text() promise
        .then(function(data){
          console.log(data);
        })
        //catch error
        .catch(function(err){
          console.log(err);
        })
      }


      //work with JSON with Fetch

      function getJSON(){
        fetch("test.json")
          .then(function(res){
            //res.json() returns a promise
            return res.json();
          })
          //fulfill the promise from .json()
          .then(function(data){
            console.log(data);
          })
          //catch error
          .catch(function(err){
            console.log(err);
          })
      }

Aysnc/ Await

The async keyword by itself generates a promise. See the code below: 

      async function myFunc(){
        console.log("Hello");
      }

      myFunc(); //logs a promise
      //if we want to log the string we do
      myFunc().then(res => console.log(res));

Inside an async function we can use the await keyword to await for an async operation (for instance for the fulfillment of a promise): 

      async function myFunc(){
         const promise = new Promise((resolve, reject)=>{
           setTimeout(resolve('Hello'), 1000)
         });

         //wait for promise
         const res = await promise;
         return res;
      }


      myFunc().then(res => console.log(res));
