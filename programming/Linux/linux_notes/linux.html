The Linux Desktop Environment

A Desktop environment is a graphical tool that displays icons and menus on the screen. It controls the appearance and provides programs/utilities that help with common tasks like file management

Among the most prominent environments we find: Unity, GNOME, KDE, Xfce

Nautilus is the main file explorer in linux (default file explorer - cabinet icon).

In the home directory we have only user related elements (NO system elements). 

      /home

In 'other locations' we can see the root directory of the system. In 'computer' we find everything on the pc. In 'other locations' we also have the 'network' button to connect to networks

Getting familiar with the command line

A lot of functionality in linux can be done through the GUI, but power users preffer the command line to perform the tasks more quickly and more efficiently. Everything in the GUI can be done through the command line.

The command line will offer more options/ways to execute a program.

We can put multiple programs in a shell script (a file containing multiple intructions that get executed 1 after another), and this is a good way to automate tasks.

We can open a terminal from the system or connect remotely with a client like Putty.

some basic commands:

ls (displays the contents of the current directory)

ls /var (displays the contents of the var directory)

pwd (print working directory-prints the path to the current directory)

cd .. (move to parent directory)

cd ../.. (move 2 directories up)

If we want to launch a program such as the firefox browser, we can do: 

      /usr/bin/firefox

We can also simply type firefox in the terminal.

the $PATH variable in Linux

The firefox command above works because of the path variable. The path variable contains the locations in which the system will look when we type a command that is not in its full path.

When we type firefox the browser starts up because the location where firefox is installed is in the $PATH variable (in /usr/bin ).

In order to see the full path variable type:

      echo $PATH

This will display the full $PATH variable:

      /home/amaimon/.deno/bin:/home/amaimon/.cargo/bin:
      /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:
      /sbin:/bin:/usr/games:/usr/local/games:/snap/bin

The directories in the $PATH variable are separated by semicolons. Note that in the variable above, the /usr/bin directory is included (that's where firefox is installed and that's how the system knows to start up the browser when we use only the firefox command).

The system will look for the directories in the $PATH variable in the following order:

first will look in usr/local/sbin, next in usr/local/bin, next usr/sbin, then in usr/bin , etc... so for instance having 2 programs installed with the same name in 2 different locations in the $PATH, the first one will run first.

Bash Scripting

In a linux distribution we can create scripts that can automate tasks for us (can run programs and perform various actions). We will create a simple one for testing purposes.

Create a test.sh file in the current directory. Open the file in a text editor and type in it: 

      echo 'Hello from Ubuntu =)!';

Save the file, and run it by typing in the terminal:

      ./test.sh

the ./ makes it so that the terminal instance runs the program from the current directory (where the test.sh script is located). If test.sh were in a different folder, we would have to type the full path to it. The output of the command above will be: 

      Hello from Ubuntu =)!

The filesystem tree in Linux

The base directory is called root /

Under the root / directory there are directories like: home /home, var /var, media /media, etc...

Under these there are the users directories home/dragos home/dragos, the logs home/log, etc...
