      /home

SSH

SSH (Secure Shell) is a communication protocol (like http, https, ftp). We can use SSH for anything (create files/folders, install/run programs, deploying applications, setting up web servers). SSH is secure (all the traffic will be encripted)

SSH Client/Server Communication

In order for us to log onto a machine with SSH the credentials are not enough, the machine must also have the SSHD (Open SSH Daemon) installed and running. The daemon listens for SSH connections.

The server must have SSHD installed and running or we will not be able to connect to it using SSH.

On the server we have an sshd configuration file (where we can change configuration such as disabling password login for the root user).

SSH Authentication Methods

Password (default way of authentication). In order to authenticate we use the following command: 

      ssh user@192.168.1.29 //can use domain or IP address

Once we do that, it will prompt us for a password (if we do not have SSH keys set up).

Public/Private Key Pair (recommended way)

Same command as above to log in and the program will let us in without asking for a password.

Host Based

Via a known hosts file which allows certain hosts to connect to a machine.

Generating Keys

The command for generating keys is:

      ssh-keygen

This command will create a public key and a private one.

The keys will be created by default in the home directory in a hidden folder (.ssh). The command will prompt you to choose a different name other than id_rsa for the keys, but this is the default.

~/.ssh/id_rsa (Private Key)

~/.ssh/id_rsa.pub (Public Key)

The public key goes on the server. On the server we will also have a .ssh folder (or will create one if it does not exist), and in that directory we'll have a file called authorized_keys (this is where the public key will be put).This is how the server knows if the machine we are using to log onto it is authorized to do so.
